# Krono — Stack e Configurações Técnicas

## Linguagem e SDK
- **Kotlin** — sem Java
- `minSdk` 26 (Android 8.0) · `targetSdk` 35 (Android 15)
- `compileSdk` 35

## UI
- **Jetpack Compose** exclusivo (sem XML layouts)
- **Material 3** (`androidx.compose.material3`)
- Sem Previews do Compose

## Programação Assíncrona
- **Kotlin Coroutines** com `StateFlow` e `SharedFlow`
- `collectAsState()` nos Composables
- `collectLatest` nos observers do `MainService`

## Persistência
| Arquivo | Mecanismo | Uso |
|---|---|---|
| `OverlayDataStore.kt` | DataStore Preferences | Configurações reativas do overlay |
| `TimerPreferences.kt` | SharedPreferences | Estado síncrono para reboot |

## Navegação
- `androidx.navigation.compose` — NavHost com rotas string (`AppRoutes.TIMER`, `AppRoutes.SETTINGS`)

## Temas (KronoTheme.kt)
```kotlin
enum class KronoThemeOption(val label: String) {
    AUTO, DARK_MODERN, SOLARIZED_DARK, LIGHT_MODERN, SOLARIZED_LIGHT
}
```
Cores principais por tema:
| Tema | Background | Primary |
|---|---|---|
| Dark Modern | `#1E1E2E` | `#7B8CDE` |
| Solarized Dark | `#002B36` | `#268BD2` |
| Light Modern | `#FFFFFF` | `#6B7FD4` |
| Solarized Light | `#FDF6E3` | `#2075C7` |

## Overlay (WindowManager)
```kotlin
WindowManager.LayoutParams(
    WRAP_CONTENT, WRAP_CONTENT,
    TYPE_APPLICATION_OVERLAY,
    FLAG_NOT_FOCUSABLE,   // removido temporariamente quando menu rápido abre
    PixelFormat.TRANSLUCENT
)
```
O `ComposeView` do overlay é envolvido em `KronoTheme` para que Material3 funcione corretamente.

## Atualização
- `UpdateChecker.kt`: consulta GitHub API `releases/latest`
- `ApkInstaller.kt`: usa `DownloadManager` (background, com progresso) + `FileProvider` (authorities: `${packageName}.provider`) + Intent `ACTION_VIEW` com `application/vnd.android.package-archive`
- APK salvo em `cacheDir` e deletado via `BroadcastReceiver(ACTION_PACKAGE_ADDED)`

## Gradle (app/build.gradle.kts)
- `versionName`: SemVer, sincronizado com `package.json` pelo script `update-version.js`
- `versionCode`: gerado dinamicamente `yyMMddHH`
- Task `extractChangelog`: extrai seção da versão atual do `CHANGELOG.md` → `res/raw/changelog.md`
