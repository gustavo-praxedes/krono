# Krono — CI/CD e Versionamento

## Padrão de Commits
**Conventional Commits** — o prefixo determina o bump de versão:

| Prefixo | Bump | Exemplo |
|---|---|---|
| `feat:` | Minor | `feat: add quick menu` |
| `fix:` | Patch | `fix: notification not updating after reset` |
| `perf:` | Patch | `perf: reduce tick from 10ms to 100ms` |
| `docs:` | — | `docs: update readme` |
| `chore:` | — | `chore: update dependencies` |
| `BREAKING CHANGE:` | Major | — |

## Ferramentas
- **`commit-and-tag-version`** — analisa commits, gera changelog, bumpa versão, cria tag
- **`npm run release`** — comando único que dispara tudo
- Configurado via `.versionrc` na raiz

## Scripts (pasta `scripts/`)

### `update-version.js`
Ponte entre `commit-and-tag-version` e o Kotlin. Usa Regex para ler e escrever `versionName` no `app/build.gradle.kts` a partir do `package.json`.

### `update-readme.js`
Executado automaticamente no hook `postbump`. Atualiza `README.md` com a versão e o último changelog antes do commit de release.

### `generate_changelog.py`
Hook `post-commit` do Git. Após cada commit:
1. Lê commits desde a última tag
2. Parseia tipos (feat/fix/perf/etc)
3. Agrupa por categoria com emoji e labels
4. Lê `versionName` do `build.gradle.kts`
5. Gera/atualiza seção no `CHANGELOG.md`
6. Faz `git commit --amend --no-edit` silencioso

**Instalação:** `.git/hooks/post-commit` chama `/e/Miniconda3/python.exe scripts/generate_changelog.py`

## Task Gradle: extractChangelog
Definida em `app/build.gradle.kts`. Roda antes de todo build:
- Lê `CHANGELOG.md` na raiz
- Encontra a seção `## [X.Y.Z]` que bate com o `versionName`
- Salva apenas aquele bloco em `app/src/main/res/raw/changelog.md`
- `AboutDialog` lê esse arquivo instantaneamente (sem rede)

## GitHub Actions (`release.yml`)
Disparado por push de tags `v*`:
1. Checkout do código
2. Setup JDK
3. Build do APK de release (`./gradlew assembleRelease`)
4. Cria GitHub Release com o APK e as notas do changelog

## Como fazer uma nova versão
```bash
# 1. Commits normais durante o desenvolvimento
git commit -m "feat: Nova funcionalidade."
git commit -m "fix: Correção de bug."

# 2. Quando pronto para release
npm run release

# 3. Push com tags
git push --follow-tags origin main

# 4. GitHub Actions gera o APK e cria a Release automaticamente
```

## Verificação de Atualização no App
- `checkForUpdateIfNeeded()` roda 1x/dia na abertura do app
- Compara `versionName` instalado com `tagName` da última release do GitHub
- Normaliza o prefixo `v` antes de comparar: `"v2.4.0".removePrefix("v") == "2.4.0"`
- Se há versão nova: `pendingUpdateInfo` é preenchido e `UpdateDialog` aparece
