# Krono — Arquitetura e Estrutura de Arquivos (v2.5+)

## Padrão Geral
MVVM com `StateFlow` + `collectAsState`. DataStore para persistência reativa. Foreground Service para background com **Arquitetura de Delegação**. Jetpack Compose para toda a UI.

---

## Estrutura de Pacotes

```
com.krono.app/
├── KronoApp.kt                  # Application: canal de notificação + singleton TimerViewModel
├── Constants.kt                 # Actions de Intent, extras, IDs de notificação
│
├── ui/
│   ├── MainActivity.kt          # Activity: ciclo de vida e orquestração de permissões
│   ├── AppNavigation.kt         # NavHost e rotas principais
│   ├── SettingsScreen.kt        # Tela de configurações internacionalizada
│   ├── SettingsComponents.kt    # Componentes de UI padronizados (ToggleRow, Slider, etc)
│   ├── FloatingTimerUi.kt       # UI do overlay flutuante
│   ├── ColorPickerDialog.kt     # Componente isolado de seleção de cores
│   ├── DonationDialog.kt        # Lógica de interação de doação
│   └── OverlayPermissionDialog.kt # Dialog de permissão de sistema
│
├── service/
│   ├── MainService.kt           # Orquestrador de background
│   ├── OverlayManager.kt        # Delegado: Único dono da WindowManager e UI flutuante
│   ├── NotificationHelper.kt    # Delegado: Construção e atualização da notificação
│   └── FeedbackManager.kt       # Delegado: Gerenciamento de Beeps e Vibrações
│
├── viewmodel/
│   └── TimerViewModel.kt        # Lógica do cronômetro. Loop otimizado: 250ms
│
├── data/
│   ├── OverlayDataStore.kt      # Persistência reativa (OverlayConfig)
│   └── TimerPreferences.kt      # Persistência síncrona para sobrevivência a reboots
```

## Responsabilidades e Decisões Técnicas

### ⏱️ Otimização de Bateria
O `TimerViewModel` utiliza um intervalo de **250ms** para atualização de estado. Esta escolha equilibra a percepção visual de fluidez com a redução drástica de recomposições e consumo de CPU, essencial para um widget que opera em overlay.

### 🏗️ Modularização do Serviço
O `MainService` não manipula mais hardware ou janelas diretamente. 
- O `OverlayManager` cuida de `TYPE_APPLICATION_OVERLAY`, arraste e flags de foco.
- O `NotificationHelper` abstrai a complexidade do cronômetro nativo na barra de notificações.

### 🌐 Internacionalização (i18n)
O projeto não aceita strings *hardcoded*. Todos os textos devem ser referenciados via `R.string` para permitir escalabilidade e suporte a múltiplos idiomas.

### 🧩 Desacoplamento de UI
Telas e modais complexos devem residir em arquivos próprios. A `MainActivity` e `SettingsScreen` devem permanecer limpas, atuando apenas como containers de lógica de estado e navegação.
